\section{Implementation and Results}\label{sec:results}

\subsection{Implementation}

The translation is implemented by our tool \texttt{eo2lp},
an OCaml program of approximately 5{,}500~lines.
%
The Eunoia parser is generated by Menhir~\cite{menhir}
and the encoding back-end is built against
the $\lambdapi$ OCaml API.
%
Processing follows a three-phase pipeline:
(i)~parsing Eunoia files into an AST and
computing a dependency-ordered module sequence,
(ii)~elaboration via the operator of
\autoref{sec:eo-elab}
(resolving overloads, expanding $n$-ary operators,
and normalizing binder applications),
and
(iii)~encoding elaborated terms as $\lambdapi$
commands via the translation operators of
\autoref{sec:lambdapi}.

The type embedding described in
\autoref{sec:lambdapi} is bootstrapped by a
hand-written \texttt{Prelude.lp} module
(${\sim}335$~lines) that declares the core type
infrastructure ($\Set$, $\msf{τ}$, function
space~$⤳$), the higher-order application symbol
$\msf{APP}$, Boolean logic, arithmetic builtins,
and the heterogeneous list type used by quantifier
binders.
%
Side-condition programs are translated to
$\lambdapi$ symbols with the \texttt{sequential}
modifier whose rewrite rules encode the
pattern-matching cases of each program,
and $\lambdapi$'s rewriting engine evaluates
them during type-checking.
%
Placeholder arguments inserted for implicit
parameters are resolved via $\lambdapi$'s
higher-order unification engine.

\subsection{CPC-MINI Signature}
%
The Cooperating Proof Calculus (CPC) is the
Eunoia signature that formalizes \cvc{5}'s proof
system.
%
Because the full CPC is large (${\sim}51$~modules)
and still evolving,
we define CPC-MINI, a subset of 21~modules
(${\sim}5{,}000$~lines of Eunoia) covering the
theories, inference rules, and side-condition programs
needed for the SMT-LIB fragments we target.
%
CPC-MINI spans 6~theory modules
(e.g.\ \texttt{Arith}, \texttt{Arrays},
\texttt{Quantifiers}),
8~rule modules
(e.g.\ \texttt{Booleans}, \texttt{Uf},
\texttt{Rewrites}),
and 7~program modules
(e.g.\ \texttt{Nary}, \texttt{AciNorm},
\texttt{PolyNorm}),
declaring 57~theory constants,
353~inference rules,
and 42~side-condition programs in total.
%
All 21~modules are translated by \texttt{eo2lp}
into $\lambdapi$ files that pass
\texttt{lambdapi check} with full subject
reduction verification.

\subsection{Proof Benchmarks}
%
For each of 18~SMT-LIB fragments supported by
CPC-MINI, we sampled up to 100~unsatisfiable
problems from the SMT-LIB benchmark library and
obtained Eunoia proof scripts from \cvc{5}.
%
Some fragments have fewer than 100~available
proofs, yielding a total of 1{,}227~proofs.
%
Each proof is translated by \texttt{eo2lp} and the
generated $\lambdapi$ file is checked by
\texttt{lambdapi check} with subject reduction
verification enabled, under a 5-second timeout.

\autoref{tab:results} summarizes the results.
%
Overall, 1{,}119 of 1{,}227 proofs
(91\%) are successfully translated and
verified, with an average time
of 0.47\,s per proof.
%
Of the 18~fragments,
8~achieve a 100\% pass rate
and a further 5~exceed 95\%.
%
The 108~failures comprise
47~type-checking errors and
61~timeouts (during either encoding or checking).

\subsubsection{Analysis of Failures.}
%
Failures are concentrated in two areas.
%
First, the NIA fragment accounts for 62 of the
108~failures (22~type-checking errors and 40~timeouts),
and QF\_NIA contributes a further 11.
%
Type-checking errors in these fragments arise from
proof steps that invoke inference rules present in
the full CPC but absent from CPC-MINI.
%
Such steps are translated using the CPC \texttt{trust}
rule, whose type is too permissive to be verified:
the generated $\lambdapi$ \texttt{assert} for the
conclusion fails.
%
Second, timeouts occur predominantly in
NIA and QF\_LIA, where the generated $\lambdapi$
files can be large and the cost of type inference
and rewriting during checking exceeds the 5-second
limit.

The 15~failures in QF\_UFLIA are type-checking
errors that stem from a small number of rules
involving interactions between uninterpreted
functions and linear arithmetic not yet covered
by CPC-MINI.
%
The remaining failures
(in LIA, QF\_UF, QF\_IDL, UFIDL, UFNIA) are
isolated, with at most 2~per fragment.


\subsubsection{Comparison with Related Work.}
%
Coltellacci et
al.~\cite{coltellacciReconstructionSMTProofs2024a}
translate Alethe proofs into $\lambdapi$ by
extending the Carcara proof checker with a
$\lambdapi$ back-end.
%
In their approach, each Alethe inference rule
corresponds to a fixed lemma in $\lambdapi$,
and proofs are reconstructed by composing these
lemmas.
%
By contrast, our translation targets Eunoia, which
is a \emph{logical framework}: the inference rules
themselves are defined within the language, not
hard-coded in the translator.
%
Consequently, \texttt{eo2lp} translates both the
proof system (CPC-MINI) and individual proof scripts,
and any extension of the rule set in CPC requires
no modification of the tool.
%
Furthermore, the side-condition programs of CPC are
encoded as $\lambdapi$ rewrite rules, so that the
computational content of rule applications is
verified by $\lambdapi$'s rewriting engine
rather than trusted.
